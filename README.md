# Завдання 1: Дослідження HTTP «вручну» через telnet/curl

## Яку версію HTTP повернув сервер у відповіді?

**У відповіді сервер повернув версію HTTP/1.1**

---

## Які заголовки є в кожній відповіді? Що означають Content-Type, Content-Length, Server?

**в кожній відповіді є такі заголовки:**

- **Date - дата і час відповіді**
- **Content-Type - тип тіла**
- **Connection - управління з'єднанням**
- **Server - назва і версія сервера**

**Content-Type - це домовленість між сервером і клієнтом який буде тип і кодування тіла.**

**Content-Length - це розмір тіла в байтах.**

**Server - це назва і версія програмного забезпечення сервера що обробив запит.**

---

## Що таке Keep-Alive? Навіщо він потрібен порівняно з HTTP/1.0?

**Keep-alive — це можливість тримати з'єднання відкритим після відповіді, щоб зробити кілька запитів через одне з'єднання.**

# Завдання 2: HTTP методи та коди відповідей

## HTTP методи

| Метод | Призначення | Ідемпотентний? |
|---|---|---|
| GET | Отримати ресурс | Так |
| POST | Створити ресурс / надіслати дані | Ні |
| PUT | Замінити ресурс повністю | Так |
| PATCH | Частково оновити ресурс | Ні |
| DELETE | Видалити ресурс | Так |
| HEAD | Як GET, але без тіла відповіді | Так |
| OPTIONS | Запитує список дозволених методів для ресурсу | Так |

---

## Статус-коди

| Код | Назва | Коли використовується |
|---|---|---|
| 200 | OK | Успішний запит |
| 201 | Created | Ресурс створено (POST/PUT) |
| 301 | Moved Permanently | Постійне перенаправлення |
| 400 | Bad Request | Невірний запит від клієнта |
| 401 | Unauthorized | Потрібна автентифікація |
| 403 | Forbidden | Відмова сервера у виконанні запиту |
| 404 | Not Found | Сервер не знайшов сторінку за вказаною адресою |
| 500 | Internal Server Error | Збій у роботі конкретного сайту |
| 502 | Bad Gateway | Помилка шлюзу. Сайт "Ліг" сервер-посередник (Nginx) не отримав відповіді від додатку |
| 503 | Service Unavailable | Сервер тимчасово не працює через технічне обслуговування або перевантаження |

---

# Різниця між 301 та 302 статус-кодом

**301 статус-код каже ця сторінка за старою адресою більше не існує і ніколи не повернеться. Вона назавжди переїхала на нову адресу**

**302 статус-код каже зараз сторінка знаходиться за іншою адресою, але це тимчасово. Скоро вона повернеться назад**

---

# Навіщо браузер кешує 301, але не кешує 302?

**Браузер кешує тому, що 301 означає що сторінка назавжди перейшла на нову адтресу тому щоб не запитувати кожного разу ми беремо відразу з кешу**

**302 не кешується бо адреса тимчасова — наступного разу вона може змінитись, тому браузер щоразу питає сервер заново**

---

# Різниця між 401 та 403 статус-кодом

**401 статус-код каже що обов'язково потрібно авторизуватися тому, що сервер не знає хто ви такий**

**403 статус-код вже знає хто ви такий але все одно не пусти вас тому, що у вас немає прав**

# Завдання 3: Структура HTTP запиту та відповіді

## HTTP Запит

```http
POST /path HTTP/version
Host: httpbin.org
User-Agent: curl/8.19.0
Accept: */*
Content-Type: application/json
Content-Length: 43

{
  "name": "Student",
  "course": "Networking"
}
```

---

## Яким роздільником розділяються заголовки від тіла запиту/відповіді?

**Заголовок від тіла розділяє пустий рядок**

---

## Що буде, якщо `Content-Type` не збігається з реальним вмістом тіла?

**Content-Type — це домовленість. Якщо вона порушена — обидві сторони працюють некоректно тобто якщо `Content-Type` не збігається з реальним вмістом тіла буде помилка**
